# Aurora 10 分钟答辩 PPT 制作资料

适用场景：2026 RoboCup 中国赛足球机器人比赛仿真 2D 组技术挑战赛答辩  
建议时长：10 分钟  
建议页数：12 页正片 + 2 页备份页  
答辩主线：规则约束下的稳定多智能体系统，规则策略建立下限，强化学习探索上限，回归评测驱动持续迭代。

## 一、整体答辩策略

10 分钟答辩不要试图展示所有细节。建议只讲清楚四件事：

1. 我们是谁：南京理工大学 Aurora 队，面向仿真 2D 组构建 11 人制足球智能体队伍。
2. 我们解决什么问题：100ms 周期内的多智能体实时协作，以及规则约束下的稳定比赛能力。
3. 我们的独特技术是什么：规则策略 + 强化学习 + fallback + 回归评测。
4. 我们下一步怎么做：继续围绕防守、后场重启、边线处理、定位球和 RL 局部增强迭代。

建议把“训练结果”放轻，只保留“当前仍在迭代，所以本次重点介绍技术路线、系统设计和可验证工程闭环”的口径。

## 二、PPT 风格建议

整体风格建议：

1. 主色使用深灰、白色、绿色或青色，贴近球场和工程感。
2. 每页只表达一个核心观点。
3. 尽量使用结构图、流程图、球场示意图，不堆大段文字。
4. 每页正文不超过 5 行，每行不超过 18 个汉字。
5. 公式只放 2-3 个关键公式，其余放备份页或文档。
6. 图片优先使用自绘示意图，不需要真实比赛截图也能讲清楚技术方案。

建议封面视觉：

```text
背景：简洁的二维足球场俯视图
中心：Aurora 队名
副标题：规则策略与强化学习融合的 RoboCup 2D 仿真足球队伍
底部：南京理工大学 / 队员与指导老师
```

## 三、10 分钟时间分配

| 环节 | 页码 | 时间 | 目标 |
|---|---:|---:|---|

### 第 6 页：核心技术一 混合控制

标题：

```text
混合控制：规则策略建立下限，RL 探索上限
```

页面内容：

```text
多数球员：规则策略控制
10 号球员：强化学习增强
18 个离散动作
RL 不可用时自动回退规则策略
```

建议配图：

```text
球场上突出 10 号球员，其余球员用规则策略标识。右侧放 18 个 action ID 的小表格。
```

可放公式：

$$
a_t=\arg\max_a \pi_\theta(a|o_t)
$$

讲稿：

```text
我们没有选择全队端到端强化学习，而是采用混合控制。大部分球员由规则策略控制，这样行为稳定且可解释；10 号球员由 RL 做局部增强。RL 的动作空间是 18 个离散动作，包括射门、盘带、传球、追球、回位和安全清球。如果模型不可用或动作超时，就自动回退规则策略。
```

时间：55-60 秒。

### 第 7 页：核心技术二 fallback 安全机制

标题：

```text
fallback：让异常不扩散成整队失控
```

页面内容：

```text
定位球异常 -> 回到阵型
踢球异常 -> 安全传球/清球
防守异常 -> 基础站位/追球
RL 超时 -> 规则策略
```

建议配图：

```text
流程图：高级决策 -> 是否成功 -> 成功执行 / 失败进入保守动作。
```

讲稿：

```text
仿真比赛里一个周期的异常可能很快变成连续失球，所以我们设计了多层 fallback。比如定位球策略异常时退回阵型移动，踢球策略异常时退回安全处理，RL 超时时退回规则策略。这样做的核心目标是保证系统在复杂比赛状态下可恢复，而不是局部失败后整队失控。
```

时间：45-50 秒。

讲稿：

### 第 9 页：核心技术四 传球评分

标题：

```text
传球评分：在推进和安全之间权衡
```

页面内容：

```text
传球路线是否安全
接球队员周围是否有空间
是否带来向前推进
是否过度靠边
后场受压时允许安全重置
```

建议配图：

```text
持球队员、候选接球队员、两名防守队员、传球路线。标注线路安全距离 M_i、接球空间 D_i、推进量 Delta x。
```

建议放公式：

$$
\operatorname{Score}(i)=2.2\Delta x_i+1.5M_i+D_i-0.08|y_i|+B_i
$$

讲稿：

```text
我们的传球不是简单找最靠前的队友，而是做安全评分。这里 Delta x 表示向前推进，M 表示传球线路到最近对手的安全距离，D 表示接球队员周围空间，y 项用于轻微惩罚过度靠边的位置，B 表示后场压力或进攻阶段带来的额外修正。最终选择满足安全约束且评分最高的目标。
```

时间：60 秒。

## 六、现场答辩口径

### 如果被问：为什么不用全队强化学习？

建议回答：

```text
全队端到端强化学习在训练样本、稳定性和可解释性上风险较高。RoboCup 2D 比赛现场首先要求球队稳定运行和规则合规，所以我们采用混合控制。规则策略保证全队下限，强化学习先在 10 号球员上做局部增强，后续再逐步扩大可学习范围。
```

### 如果被问：你们的独特性在哪里？

建议回答：

```text
我们的独特性不只是用了 RL，而是把规则策略、RL、fallback 和回归评测组合成闭环。规则策略负责可解释协作，RL 负责探索局部动作选择，fallback 负责异常恢复，固定对手池负责验证每次修改是否有效。
```

### 如果被问：当前最需要改进的地方？

建议回答：

```text
当前主要改进方向是强对抗下的防守组织，包括边线争抢、后场重启、禁区保护、第二点球处理和定位球防守。我们会先把防守稳定性和失球链路压下来，再进一步补充进攻和定位球套路。
```

### 如果被问：训练结果为什么不展开？

建议回答：

```text
目前策略仍在持续迭代，训练和回归结果还会随版本变化。为了避免展示不稳定的阶段性数字，本次重点介绍系统架构、技术路线和评测方法。等候选版本冻结后，我们会加入完整对手池结果和趋势图。
```

### 如果被问：如何保证符合 2026 规则？

建议回答：

```text
我们按照 Ubuntu 24.04 和 rcssserver-19.0.x 环境准备运行链路，球队主程序中集成了标准赛事身份信息输出。同时策略设计遵守服务器通信机制，不使用越界通信或其他队伍可执行代码，提交版本会保持底层和审核材料一致。
```
