# RoboCup 一天激进提分策略

## 当前状态分析

已测试27个候选方案（candidate_25-51），全部未达到20%提升目标。当前基线（candidate_12）每轮失球约96-100个。最佳尝试（candidate_47）仅提升14.68%，距离20%目标差5.32%。

**当前失球分布（candidate_50/51基线）：**
- **flank_kickoff_entry（边路开球进攻）**: 31-33球 (32-33%) — **最大失球源**
- **flank_setplay_entry（边路定位球进攻）**: 17-20球 (18-20%) — **第二大失球源**
- **center_kickoff_entry（中路开球进攻）**: 16-21球 (17-22%) — **第三大失球源**
- **center_midfield_gap（中路中场空档）**: 12-13球 (13%)
- 其他所有类型: <10球

**核心问题：** 每次防守改进都会产生补偿性退化。修复开球结构会增加终端进球。添加终端防守会把门将拉到前场（`nearest_our_band=goalie +21`）。决策逻辑无法同时防守多个进攻阶段。

## 失败方案分析

### 模式1：终端防守毒性（candidates 36, 38, 41-42, 44-46, 48）
**尝试内容：** 添加门柱防守、后卫线架子、球门口锁定、终端清球
**结果：** 持续增加 `nearest_our_band=goalie` +21-56球
**失败原因：** 终端行为把防守球员（尤其是门将）拉到激进位置，导致球门暴露

### 模式2：开球第一触球操控（candidates 28-29）
**尝试内容：** 安全大脚到对方角落、边路出球给#9/#11
**结果：** 大幅退化 — `kick_off`失球从37-53增加到65-69
**失败原因：** 错误的执行杠杆；控球比初始球位置更重要

### 模式3：结构性站位（candidates 33-34, 37, 39-43, 47）
**尝试内容：** 反抢、通道锁定、中路通道锚定、定位球墙、前线恢复屏障
**结果：** 混合 — 减少了结构性失球但失球迁移到终端进攻
**最佳表现：** Candidate_47（14.68%提升）完全移除了门柱/终端防守
**部分成功原因：** 结构性站位而不激进终端防守是正确方向
**未达标原因：** `center_kickoff_entry`变得脆弱（+12球）

### 模式4：上下文门控（candidates 40, 50-51）
**尝试内容：** 对手特定门控（仅Cyrus）、严格开球上下文（非开球重启后禁用）
**结果：** 小幅提升（5-6%）但不足
**失败原因：** 太窄；未解决根本协调问题

### 模式5：中场空档填补（candidates 25, 31）
**尝试内容：** 中路中场空档填补、门将禁区清道夫
**结果：** 适得其反 — `center_midfield_gap`增加+8-15球
**失败原因：** 填补旧空档时创造了新空档

### 策略4（疯狂射门）的数学

```python

```

这些条件在实战中**几乎不可能同时满足**。即使满足，Aurora也不敢射。

策略4放宽到：
```python
max_distance = 35.0  # 35m内都能射（半场）
max_angle = 60.0     # 60度内都能射（侧面）
min_me_x = 15.0      # 过半场就能射
```

- 每场比赛Aurora控球时间：30%
- 在对方半场时间：15%

- 每30周期射一次 = 10次射门/场
- 5%命中率 = 0.5球/场

- 远射造成门将失误

**保守估计：3-5球/场**
